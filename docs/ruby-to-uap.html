7. DSL 仕様 v0


    

      
対応構文一覧

      

        
          

            記法
            変換後命令
          

        
        
          

            # comment
            無視
          

          

            v = GPIO.new(3, GPIO::OUT)
            GPIO_MODE pin=3 OUTPUT
          

          

            v = GPIO.new(9, GPIO::IN)
            GPIO_MODE pin=9 INPUT
          

          

            v = GPIO.new(9, GPIO::IN | GPIO::PULL_UP)
            GPIO_MODE pin=9 INPUT_PULLUP
          

          

            v = PWM.new(2)
            GPIO_MODE pin=2 OUTPUT
          

          

            v.write(1) / v.write(0)
            GPIO_WRITE pin=N value=1/0
          

          

            v.frequency(440)
            PWM_FREQ pin=N freq=440
          

          

            v.frequency(0)
            PWM_FREQ pin=N freq=0（停止）
          

          

            wait_ms 100 / wait_ms(100)
            WAIT_MS 100
          

          

            tone_ms v, 440, 200
            PWM_FREQ 440 → WAIT_MS 200 → PWM_FREQ 0
          

          

            loop 5 do ... end
            ボディを 5 回展開（ジャンプなし）
          

          

            loop do ... end
            JUMP で先頭へ戻る（無限ループ）
          

        
      


      

        v0 では if / while / each / def / クラス / 配列 / 浮動小数点 には対応しません。

        将来拡張のため parseDsl → compileAst → encodeUap の 3 段階で実装されています。
      

    

  


  
  

    
8. UIAPduino スケッチ


    

      

        

          
UapRunner.ino

          

            
            
            GitHub ↗
            
          

        

        

      

      

        FQBN: UIAP_HID:ch32v:CH32V003:pnum=V14,usb=webhid,opt=oslto
         ／  SDmin.h はボードパッケージに含まれます。
      

      

        SD カードに SCRIPT.UAP を書き込んだ後、WebHID 接続して
        0x10 RUN コマンドを送信すると TinyVM が逐次実行します。

        v0 ではブラウザからの RUN ボタンは未実装です。SD への書き込みと確認を最優先にしています。

Ruby 風 DSL

wait_ms / loop

        

        
→

TinyVM 命令変換
UAP1 生成

        

        
→

→

14 byte チャンク

        このページは Ruby 処理系をブラウザや UIAPduino 上で実行しません。

        Ruby 風 DSL を独自バイトコードへ変換します。

        UIAPduino 側では TinyVM が .uap を 1 命令ずつ読み取って実行します。
